## 7.1 Implications for Government AI Deployment

### 핵심 메시지

본 연구의 가장 중요한 시사점은 다음과 같다:

```
┌─────────────────────────────────────────────────────────────────┐
│                     CORE IMPLICATION                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  "VLM의 범용 지능(General Intelligence)을 과신하면 안 된다."     │
│                                                                  │
│  영어 벤치마크에서 90% 달성한 모델이                             │
│  일본어 공문서에서는 26%로 추락할 수 있다.                       │
│                                                                  │
│  → 정부 AI 도입 전, 대상 지역/언어에서 반드시 검증해야 한다.    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 정부 AI 도입 시 실용적 권고사항

**1. 배포 전 지역별 검증 필수**

| 권고 | 설명 |
|------|------|
| 대상 지역 데이터로 별도 평가 | 영어 벤치마크 성능만으로 판단 금지 |
| 실제 업무 문서 샘플링 | 실험실 데이터가 아닌 현장 데이터 사용 |
| 최소 성능 기준 설정 | 예: "정확도 70% 미만이면 도입 불가" |

**2. 지역별 최적화 전략**

```

│  └─ GPT 계열 모델 도입 시 주의 (33.2% 성능)                     │

│  └─ 다국어(EN/FR/DE) 일관성 검증 권장                            │

│                                                                  │
```

**3. 고부담(High-Stakes) 업무에서의 Human-in-the-Loop**

| 업무 유형 | AI 역할 | Human 역할 | 권고 |
|-----------|---------|------------|------|
| 민원 자동 분류 | 1차 분류 | 검토/확정 | AI 보조 OK |
| 시험 자동 채점 | 채점 초안 | 최종 확정 | 주의 필요 |
| 법률 문서 해석 | 참고 의견 | 최종 판단 | AI 단독 금지 |
| 행정 결정 | 보조 | 최종 결정 | AI 단독 금지 |

### 정책적 시사점

1. **AI 도입 평가 프레임워크 필요**: 각국 정부는 AI 시스템 도입 전 언어/지역별 성능 검증을 의무화하는 프레임워크 마련 필요

2. **벤더 성능 보고 요구**: AI 서비스 제공자에게 영어뿐 아니라 대상 언어에서의 성능 보고 요구

3. **단계적 도입 권고**: 저위험 업무 → 중위험 업무 → 고위험 업무 순으로 단계적 도입

## 7.2 Why Do Regional Gaps Exist?

### 지역 격차의 원인 분석

```
┌─────────────────────────────────────────────────────────────────┐
│                  ROOT CAUSE ANALYSIS                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  CAUSE 1: 학습 데이터 불균형 (Training Data Imbalance)          │
│  ─────────────────────────────────────────────────               │
│  • 대부분의 VLM 학습 데이터는 영어 중심                          │
│  • 아시아 언어 텍스트-이미지 쌍 데이터 부족                      │
│  • 공무원 시험/정부 문서는 학습 데이터에 거의 없음               │
│  • 결과: 영어 문서 처리 능력 >> 아시아 문서 처리 능력           │
│                                                                  │
│  CAUSE 2: OCR 품질 격차 (OCR Quality Gap)                       │
│  ─────────────────────────────────────────────────               │
│  • VLM 내장 OCR은 라틴 문자에 최적화                             │
│  • 복잡한 문자체계(한자, 한글, 데바나가리)에서 품질 저하         │
│  • Gemini: Google Lens 기술로 다국어 OCR 강점                   │
│  • GPT: 아시아 언어 OCR 품질 상대적 약점                         │
│  • 결과: 같은 시험이라도 문자체계에 따라 성능 편차               │
│                                                                  │
│  CAUSE 3: 관할권 지식 부족 (Jurisdiction Knowledge Gap)         │
│  ─────────────────────────────────────────────────               │
│  • 미국법/유럽법 지식 >> 한국법/일본법 지식                      │
│  • 각국 행정 체계에 대한 지식 불균형                             │
│  • 지역 특수 용어/개념 이해 부족                                 │
│  • 결과: 지역별 도메인 지식 격차                                 │
│                                                                  │
│  CAUSE 4: 평가 벤치마크 편향 (Benchmark Bias)                   │
│  ─────────────────────────────────────────────────               │
│  • 기존 벤치마크 대부분 영어 중심                                │
│  • 아시아 언어 벤치마크 부족 → 개선 동기 부족                    │
│  • "MMMU 90% 달성" ≠ "다국어 능력 우수"                          │
│  • 결과: 영어 성능은 개선되지만 다국어 성능은 정체               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 개선 방향 제안

**단기 (1-2년)**
- 아시아 언어 특화 OCR 전처리 파이프라인 개발
- 지역별 fine-tuning 데이터셋 구축
- 모델 선택 시 대상 언어 성능 기준 적용

**중기 (2-5년)**
- VLM 학습 데이터에 다국어 문서 비중 확대
- 다국어 벤치마크(EuraGovExam 포함) 정기 평가 체계 구축
- 정부 문서 특화 모델 개발

**장기 (5년+)**
- 언어 불가지론적(Language-Agnostic) VLM 아키텍처 연구
- 문자체계 복잡도에 강건한 시각 인코더 개발
- 글로벌 정부 AI 성능 표준 수립

## 7.3 Broader Implications

### AI Fairness and Equity

```
┌─────────────────────────────────────────────────────────────────┐
│                  EQUITY IMPLICATIONS                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  현 상황:                                                        │
│  • 영어권 국가: 63.7% (EU 기준) - AI 도입 가능 수준              │
│  • 아시아권 국가: 26-38% - AI 도입 시 심각한 품질 문제          │
│                                                                  │
│  시사점:                                                         │
│  • AI 기술의 혜택이 불균등하게 분배됨                            │
│  • 영어권 국가는 AI 자동화 혜택을 누리는 반면                    │
│  • 비영어권 국가는 동일 수준의 혜택 불가                         │
│  • → "디지털 식민주의(Digital Colonialism)" 우려                │
│                                                                  │
│  필요한 조치:                                                    │
│  • 다국어 AI 연구에 대한 투자 확대                               │
│  • 비영어권 지역 데이터 수집/구축 지원                           │
│  • 글로벌 AI 성능 격차 모니터링 체계 구축                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Trust in AI Systems

- 본 연구는 VLM의 **신뢰성(Trustworthiness)** 문제를 제기
- "동일 모델, 동일 과제"인데 지역만 바뀌면 성능이 3.9배 차이
- → AI 시스템에 대한 신뢰 구축을 위해 투명한 성능 보고 필요

### Benchmark Design

- 기존 벤치마크의 한계 노출: 영어 중심, 깨끗한 이미지, 보편적 지식
- EuraGovExam: 다국어, 실제 스캔 문서, 관할권별 지식
- → 더 현실적인 벤치마크가 더 의미 있는 평가를 제공

## 7.4 Limitations

### 1. Geographic Coverage

```
포함된 지역: 한국, 일본, 대만, 인도, EU (5개)
미포함 지역: 동남아, 중동, 아프리카, 중남미

한계점:
• 세계 인구의 상당 부분이 미포함
• 아랍어, 타밀어, 스와힐리어 등 미평가
• 일반화 범위에 제한

향후 연구:
• 추가 지역/언어 확장
• 특히 저자원 언어(Low-Resource Languages) 포함
```

### 2. Data Contamination

```
우려 사항:
• 일부 오래된 시험 문제는 인터넷에 공개됨
• VLM 학습 데이터에 포함되었을 가능성

대응 조치:
• 연도별 성능 분석 실시 (2023+ vs 이전)
• Temporal clean subset 정의
• 최신 시험(2023-2024)에서도 동일 패턴 확인

한계:
• 완전한 contamination 배제는 불가능
• 그러나 핵심 발견(Nation > Task)은 연도와 무관하게 일관적
```

### 3. OCR Dependency

```
우려 사항:
• 4-Track 분석에서 사용한 OCR 엔진의 품질에 의존
• OCR 엔진 자체의 편향이 결과에 영향 줄 수 있음

대응 조치:
• 상용 수준의 OCR 엔진 사용 (Google Vision API)
• Oracle text (인간 타이핑)와 비교 분석

한계:
• 다른 OCR 엔진 사용 시 결과가 달라질 수 있음
• 그러나 Track A (Image Only) 결과는 OCR과 무관
```

### 4. Causality

```
본 연구의 성격:
• 관찰 연구(Observational Study)
• 인과관계(Causality)가 아닌 상관관계(Correlation) 보고

한계점:
• "복잡한 문자체계 → 낮은 성능"의 인과관계 확정 어려움
• 다른 교란 변수(시험 난이도, 문화적 차이) 존재 가능

대응:
• 통제 실험(Controlled Experiment) 설계로 일부 보완
• 동일 시험, 다른 모델 비교로 시험 난이도 통제
• 그러나 완전한 인과 추론은 향후 연구 과제
```

### 5. Model Selection

```
포함된 모델: 23개 (Closed 9개, Open 14개)
미포함 모델: 일부 최신 모델, 지역 특화 모델

한계점:
• 모든 VLM을 평가하지 못함
• 중국 내 모델(Baichuan, Ernie 등) 미포함
• 일본/한국 특화 모델 미포함

향후 연구:
• 지속적인 모델 추가 평가
• 지역 특화 모델과의 비교
```

### 6. Task Scope

```
포함된 과목: 17개 (법학, 행정, 경제, 과학 등)
포함된 형식: 객관식(4-5지선다)

한계점:
• 주관식, 서술형 문제 미포함
• 실제 업무는 객관식보다 복잡
• 문서 요약, 정보 추출 등 미평가

향후 연구:
• 주관식/서술형 평가 추가
• 실제 정부 업무 시뮬레이션
```
